# 一、项目名称
slave-beasts

------------------------------------------------------------
# 二、核心规则（最重要）
1. **每个 Agent 同一时间只能接收和执行一个任务**
2. 若Agent忙碌，新任务必须进入该Agent的独立排队队列
3. 只有当前任务完全完成，才会从队列取下一个任务执行
4. 整体流程依然遵循：
   老细 → 包工头 → 桥王 → 天文台 → 包工头 → 蛇头 → 驴仔 → 忍者神龟 → 蛇头 → 包工头 → 老细
5. 所有聊天记录存入Redis，网页加载历史
6. 看板显示每个Agent：状态、当前任务、排队数量、进度

# 四、Agent 核心约束（必须实现）
每个Agent都必须满足：
- busy: true/false  → 是否正在执行任务
- queue: []         → 任务排队队列
- 同一时间只能有一个任务处于 processing
- 完成后自动取下一个任务
- 不允许并行处理多个任务
- 不允许中断当前任务

   - slavebeasts:agent:qd:queue

   - slavebeasts:agent:ceo:status
   - slavebeasts:agent:cro:status
   ...
4. 全局任务列表：slavebeasts:tasks
   
------------------------------------------------------------
# 七、Agent 状态含义
- idle：空闲，可接收新任务
- processing：执行中，**不接收新任务，新任务进入排队**
- finish：任务完成，自动取下一个排队任务
- reject：审核驳回
